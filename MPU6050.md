## Pinout
| MPU6050 Pin | Arduino Pin | Why |
|------|------|------|
| VCC | 3.3V or 5V | Power |
| GND | GND | Ground |
| SCL | A5 (Uno) / SCL | I2C clock |
| SDA | A4 (Uno) / SDA | I2C data |
| XDA | - | **Unused** (for daisy-chaining another I2C sensor) |
| XCL | - | **Unused** (for daisy-chaining another I2C sensor) |
| ADO | - | **Unused** (0V => address 0x68; 5V => address 0x69; — only needed with two MPU-6050s on the same I²C wire) |
| INT | - | **Unused** (Interupt-pin ???) |

## Register Addresses
>[!Note]
>Artificiële intelligentie werd gebruikt om de tabel van "register adresses" op te stellen. Zie [Bronnen](#bronnen) voor de door AI geraadpleegde bronnen.

|I²C adress|Internal register|
|--|--|
|0x00–0x0C|Sensor Data Internal Registers (Reserved)|
|0x0D–0x0F|Self-Test Registers (XA_TEST, YA_TEST, ZA_TEST)|
|0x10–0x12| Self-Test Registers (XG_TEST, YG_TEST, ZG_TEST)|
|0x13–0x18| Reserved|
|0x19| SMPLRT_DIV (Sample Rate Divider)|
|0x1A| CONFIG (DLPF configuration)|
|0x1B| GYRO_CONFIG (Gyroscope scale: ±250–2000°/s)|
|0x1C| ACCEL_CONFIG (Accelerometer scale: ±2g–16g)|
|0x3A| INT_STATUS (Interrupt status)|
|0x3B–0x3C| ACCEL_XOUT_H/L|
|0x3D–0x3E| ACCEL_YOUT_H/L|
|0x3F–0x40| ACCEL_ZOUT_H/L|
|0x41–0x42| TEMP_OUT_H/L|
|0x43–0x44| GYRO_XOUT_H/L|
|0x45–0x46| GYRO_YOUT_H/L|
|0x47–0x48| GYRO_ZOUT_H/L|
|0x6A| USER_CTRL (Enable FIFO, I2C Master, DMP)|
|0x6B| PWR_MGMT_1 (Wakes up the chip, select clock source)|
|0x75| WHO_AM_I (Device ID - usually 0x68) |

<br>

**Essential Setup Sequence**
1. Wake Up (0x6B): Write 0x00 to PWR_MGMT_1 to wake the sensor.

2. Config (0x1A): Set Digital Low Pass Filter (e.g., 0x03).

>[!Note]
>Register names ending in _H and _L contain the high and low bytes respectively, forming a 16-bit signed integer for each axis.

  uint8_t GYRO_XOUT_L = Wire.read(); // Receive second byte = GYRO_XOUT_L data

  int16_t gyro_xout = (GYRO_XOUT_H << 8) | GYRO_XOUT_L; // Shift high byte with 8 bits to the left... Then use "or" to combine with the lower byte
